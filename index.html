Section 1 & Section 2

Khi truy cập 1 trang web (VD:https://geekadventure.vn) trình duyệt sẽ làm gì?

Trình duyệt sẽ truy cập đến DNS server và tìm địa chỉ thực của server phía website. Tiếp theo, trình duyệt sẽ gửi Http request cho server và yêu cầu server gửi lại một bản copy của website cho phía client thông qua TCP/IP. Nếu như phía server chấp nhận request từ client, server sẽ bắt đầu gửi những tập tin của website cho phía trình duyệt dưới dạng một chuỗi những khối nhỏ được gọi là gói dữ liệu. Trình duyệt sẽ tập hợp tất cả những khối dữ liệu thành một trang web hoàn chỉnh và hiển thị nó cho người dùng.

Nếu có 2 CSS selectors cùng ứng với 1 element thì style cho element sẽ thể hiện như thế nào?

  • Style cho element sẽ thể hiện dưới sự kết hợp của cả 2 CSS selectors.
  • Trong trường hợp có sự xung đột giữa các thuộc tính trong lúc triển khai, việc thuộc tính của Selector nào được thể hiện vào style sẽ căn cứ trên độ quan trọng. Độ quan trọng hay trong tiếng Anh được gọi là Importance bao gồm 5 cấp độ từ lớn đến nhỏ: User !important declarations; Author !important declarations; Author declarations; User declarations và Default browser declarations.
  • Nếu như cả 2 Selectors đều có cùng độ quan trọng thì sẽ xét đến tính Specificity. Specificity bao gồm 4 cấp độ từ lớn đến nhỏ: Inline Styles; IDs; Classes, pseudo-classes, attribute; Elements, pseudo-elements.
  • Trường hợp cuối cùng nếu như cả 2 đều có cùng mức độ specificity thì dòng khai báo cuối cùng trong code sẽ đè lên tất cả các dòng khai báo khác và được triển khai hiển thị tới Users.

Callback và Promise trong Javascript dùng để làm gì?

  • Trong Javascript, Callback là một hàm sẽ được thực thi sau khi một hàm khác kết thúc execution. Callback function thực sự rất cần thiết đặc biệt trong quá trình Asynchronous. Thông thường, Javascript sẽ được thực thi từ synchronous có nghĩa là đoạn code ở sau chỉ được thực thi trong trường hợp đoạn code ở trước đã hoàn thành. Vì vậy việc thực thi code asynchronous sẽ giải quyết tình trạng ở một thời điểm nào đó, một dòng code phải mất nhiều thời gian để hoàn thành nhưng những dòng code sau vẫn được triển khai. Callback function thường xuyên được sử dụng trong quá trình asynchronous nhằm mục đích triển khai các toán tử mà không ảnh hưởng đến các dòng code phía sau.
  • Promise trong Javascript thường được dùng để fetch dữ liệu thông qua API một cách asynchronous. Trong một vài trường hợp, sẽ mất khá nhiều thời gian để có thể fetch dữ liệu từ phía server và hiển thị lên trang web cho người dùng, việc sử dụng promise sẽ giúp người dùng có thể tiếp tục xem các nội dung khác trong khi đợi dữ liệu được fetch về.
